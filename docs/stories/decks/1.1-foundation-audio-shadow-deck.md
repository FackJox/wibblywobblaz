# Story 1.1: Foundation - Audio Architecture & Shadow Deck Core

## Status
Draft

## Story

**As a** DJ,
**I want** a visual DJ interface with dual decks that controls multiple audio sources,
**so that** I can mix music from local files, YouTube, and SoundCloud in one interface.

## Acceptance Criteria

1. Two visual turntables with play/pause, pitch control, and position scrubbing
2. WebAudio API context properly initialized for local MP3 playback
3. YouTube IFrame API integrated with volume/play control for Deck A
4. SoundCloud Widget API integrated with volume/play control for Deck B
5. Crossfader visually updates and controls volume balance between decks
6. 3-band EQ knobs per deck (visual only for streaming, functional for local)
7. BPM display shows detected/estimated tempo per track
8. Drag-and-drop zone accepts local MP3 files
9. Memory management prevents audio context leaks

## Integration Verification

- IV1: Existing routes (/home, /about, etc.) remain accessible and functional
- IV2: Page performance maintains 60fps when DJ Deck is idle
- IV3: No console errors when navigating between DJ Deck and existing pages

## Dev Notes

### Project Structure
Based on the architecture document, the DJ feature will be organized as:
```
/app/dj/                    # New route for DJ Deck
  page.tsx                  # Main DJ interface page
  layout.tsx               # DJ-specific layout
/components/dj/            # DJ-specific components
  DJDeckInterface.tsx     # Main interface orchestrator
  Turntable.tsx           # Deck visualization
  Crossfader.tsx          # Mixing control
  BPMDisplay.tsx          # Tempo indicator
/lib/dj/                   # Business logic
  audio-engine.ts         # WebAudio management
  source-adapters/        # Audio source interfaces
    LocalSource.ts
    YouTubeSource.ts
    SoundCloudSource.ts
```

### Technology Stack
- **Framework**: Next.js 15.2.4 with App Router
- **UI**: React 19, TypeScript 5
- **Styling**: Tailwind CSS 3.4.17 (existing config)
- **Components**: Radix UI (existing in project)
- **Animation**: Framer Motion 11.18.2 (existing)
- **Audio**: WebAudio API, YouTube IFrame API, SoundCloud Widget API

### Integration Points
- Must extend existing Radix UI components (Slider for crossfader/pitch)
- Use existing Tailwind configuration and color palette
- Import glitch effects from story-3.1 implementation (prepare for future BPM sync)
- Feature flag `DJ_DECK_ENABLED` in environment variables

### Key Design Specifications
**Mobile-First Sizing:**
- Turntables: 150px diameter (mobile), 250px (desktop)
- Crossfader: 80px tall × 40px wide vertical (mobile)
- Touch targets: minimum 44px for all interactive elements
- BPM Display: 36px font (mobile), 60px (desktop)

**Color Palette:**
- Primary: #8B5CF6 (active controls)
- Secondary: #EC4899 (cue buttons - future)
- Surface Dark: #1F2937 (background)
- BPM Colors: Blue (#3B82F6), Green (#10B981), Orange (#F59E0B), Red (#EF4444)

### Performance Requirements
- Maintain 60fps during all animations
- Audio latency < 100ms for local files
- Initial page load < 2 seconds on 4G
- Proper cleanup of audio contexts to prevent memory leaks

### Browser Compatibility
- Primary: Chrome, Firefox, Safari (desktop & mobile)
- WebAudio API required for local file playback
- Graceful degradation for unsupported features
- Show "Modern browser required" message if needed
