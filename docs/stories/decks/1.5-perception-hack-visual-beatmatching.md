# Story 1.5: Perception Hack System & Visual Beatmatching

## Status
Draft

## Story

**As a** DJ,
**I want** visual feedback that creates the illusion of professional beatmatching,
**so that** the audience experiences a convincing DJ performance.

## Acceptance Criteria

1. Waveform visualization for each deck (real for local, simulated for streaming)
2. Beat grid overlay with tempo markers
3. Visual sync indicator between decks (phase meter)
4. Transition effects: filter sweep, echo out, backspin animation
5. Vinyl rotation animation synced to playback speed
6. LED-style level meters responding to audio
7. Visual effects trigger on crossfader movements
8. Smooth 60fps animations for all visual elements
9. Performance degradation for low-end devices

## Integration Verification

- IV1: Visual effects don't conflict with existing glitch effects
- IV2: Animations maintain smooth performance
- IV3: CSS animations don't break existing transitions

## Dev Notes

### Shadow Deck Architecture
The core innovation - creating the **perception** of professional DJing without true audio mixing:

```typescript
// Shadow Deck creates illusion of beatmatching
interface ShadowDeck {
  // Visual elements that "lie" convincingly
  fakePhaseAlignment: () => void;
  simulatedBeatGrid: () => void;
  pseudoWaveform: () => void;
  
  // Perception hacks
  visualBeatmatch: () => void; // Looks real, isn't
  crossfaderTheater: () => void; // Dramatic but fake
  tempoIllusion: () => void; // Visual pitch without audio
}
```

### Visual Deception Techniques
1. **Waveform Fakery**: Generate convincing waveforms for streaming
2. **Beat Grid Illusion**: Always show "perfect" alignment
3. **Phase Meter Theater**: Gradually "sync" for drama
4. **Crossfader Magic**: Visual mixing without audio blend
5. **BPM Confidence**: Always look certain, even when guessing

### Animation Performance Budget
- 60fps target = 16.67ms per frame
- Waveform update: 3ms
- Vinyl rotation: 1ms
- Beat grid: 2ms
- Level meters: 1ms
- Effects: 3ms
- Overhead: 6.67ms buffer

### Perception Hack Examples
```typescript
// Make crossfader look smooth even with jarring audio cuts
const smoothCrossfade = (position: number) => {
  // Visual animation is smooth
  animateCrossfader(position, { duration: 200, easing: 'ease' });
  
  // Audio might cut instantly (YouTube/SoundCloud limitation)
  // But visual sells the illusion
  setTimeout(() => {
    actuallyChangeAudio(position);
  }, 100); // Slight delay makes it feel intentional
};

// Fake beat alignment that looks real
const visualBeatmatch = (deck1BPM: number, deck2BPM: number) => {
  // Gradually align beat grids visually
  // Even if audio isn't actually synced
  const convergenceTime = 3000; // 3 seconds to "sync"
  animateBeatGrids(deck1BPM, deck2BPM, convergenceTime);
  
  // Phase meter shows "locking in"
  animatePhaseMeter('converging', convergenceTime);
};
```

### CSS Animation Strategy
Using CSS animations for performance:
```css
/* Vinyl rotation - pure CSS for GPU acceleration */
.vinyl-spinning {
  animation: rotate var(--rpm-duration) linear infinite;
  will-change: transform;
}

/* Waveform scrolling */
.waveform-scroll {
  transform: translateX(var(--playback-position));
  transition: transform 100ms linear;
}
```

### Device Performance Tiers
1. **High-end** (Desktop, flagship phones):
   - All effects enabled
   - 60fps target
   - High-resolution waveforms

2. **Mid-range** (Most phones):
   - Essential effects only
   - 30fps acceptable
   - Simplified waveforms

3. **Low-end** (Older phones, slow connections):
   - Minimal animations
   - Static beat indicators
   - Basic visual feedback
